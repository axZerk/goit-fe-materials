Scope

Scope в JavaScript это набор переменных, обьектов и функций которые доступны в текущий момент(в текущем контексте кода). Scope JavaScript - функциональный, это значит он меняется внутри каждой функции.

Global scope

			// Scope A: Global scope out here
			var name = 'Todd';

Local scope

		// Scope A: Global scope out here
			var myFunction = function () {
				// Scope B: Local scope in here
			};

Function scope

			// Scope A
			var myFunction = function () {
				// Scope B
				var myOtherFunction = function () {
						// Scope C
				};
			};

Обьекты

Обьект - это неупорядоченная коллекция свойств, ассоциативный массив: структура, пригодная для хранения любых данных в формате ключ-значение.

В JavaScript есть три категории объектов:

  • Объекты базового типа – это объекты, определённые в спецификации ECMAScript. Например, объекты типа Array, Function, Date или RegExp являются объектами базового типа.
  • Объекты среды выполнения – это объекты, определённые в среде выполнения (такой как браузер). Например, объекты типа HTMLElement, являются объектами среды выполнения.
  • Пользовательские объекты – это любой объект, созданный в результате выполнения программного кода JavaScript.

Создание объекта

Литерал объекта – это заключённый в фигурные скобки список из нуля или более свойств (пар ключ: значение), разделённых запятыми. Именем свойства может быть любой допустимый идентификатор, строковой литерал (допускается использовать пустую строку) или число. Числовые имена свойств автоматически преобразуются в строки. Значением свойства может быть значение любого типа или выражение (значением свойства в этом случае станет результат вычисления выражения). 

		var dog = {
			name: 'Mango',
			age: 2,
			isHappy: true
		};

Синтаксис:

				var dog = {

Объекты: hasOwnProperty

Метод hasOwnProperty() возвращает логическое значение, указывающее имеет ли объект к которому вы его вызываете свойство с именем аргумента. Этот метод не ищет в цепочке прототипов объекта.

		var dog = {
			name: 'Mango',
			age: 2
		};
		for(var key in dog) {
			if(dog.hasOwnProperty(key)) {
				console.log(key); // name, age
			}
		}

Объекты: методы

  • Методы обьекта это функции обьявленые внутри объекта.
  • Методы хранятся в свойсвах объекта как function definitions
  • Для доступа к полям обьекта используется имя объекта или ключевое слово this
		var dog = {
			name: 'Mango',
			bark: function() {
				console.log('woof!');
			},
			getName: function() {
				console.log(dog.name);
			}
		};

Передача по значению и по ссылке

По значению - строки, числа, булевы значения, null/undefined при присваивании переменных копируются целиком или, как говорят, «по значению».

			var a = 5;
			var b = a;
			console.log(b); // 5
			a = 10;
			console.log(b); // 5

По ссылке - переменной, которой присвоен объект, хранится не сам объект, а «адрес его места в памяти», иными словами – «ссылка» на него.

		var obj = {name: 'Mango'};
		var val = obj;
		console.log(val); // {name: 'Mango'}
		obj.age = 2;
		console.log(val); // {name: 'Mango', age: 2}

Ключевое слово this

Контекст выполнения функции — это одно из фундаментальных понятий в JavaScript.

Контекстом еще часто называют значение переменной this внутри функции. this не привязывается к объекту, а зависит от контекста вызова и будет определено в момент вызова функции. При вызове функции для нее создается невидимый для нас обьект в котором хранятся все видимые этой функции переменные и их значения, this на него ссылается.

Function chaining

Цепочки методов, также известные как каскадирование, это многократный вызов одного метода за другим на объекте в одной непрерывной строке кода. Этот метод изобилует в jQuery и других библиотеках JavaScript.

		var dog = {
			str: '',
			addName: function() {
				this.str += 'Mango ';
				return this;
			},
			addAction: function() {
				this.str += 'says woof!';
				return this;
			},
			addSmile: function() {
				this.str += ' ^_^';
				return this;
			}
		};

		dog.addName().addAction().addSmile();
		console.log(dog.str); // Mango says woof! ^_^
