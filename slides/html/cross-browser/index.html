Кросс-браузерность

Кроссбраузерная верстка – это верстка сайта, нацеленная на создание сайта, отображаемого в каждом браузере одинаково корректно.

Как верстать кроссбраузерно?

Есть некоторые подходы и правила, которых стоит придерживаться при разработке. При условии их использования количество различий между отображением страницы в разных браузерах сводится к минимуму или исчезает совсем. Давайте рассмотрим их детальнее.

Значение свойств по умолчанию

В каждом браузере это значение может быть разным. Свойства – это цвета, шрифты, их размер. Все это должно быть описано в таблице стилей, в противном случае браузер будет самостоятельно проставлять значения, указанные по умолчанию. Поэтому в каждом из них сайт будет отображаться немного по разному.

Используйте

  • reset.css
  • normalize.css
  • Foundation
  • или хотя бы * { p0; m0; bz; }

Валидность кода

Не валидный код ведет к ошибкам отображения елементов страницы. Проверяйте ваш HTML и CSS код валидаторами.

  • https://validator.w3.org/
  • https://jigsaw.w3.org/css-validator/

Dry(don't repeat yourself) code

Пишите dry(сухой) код. Что это значит? Не используйте лишние css свойства которые ничего не делают для вашего блока. Следите за тем чтобы свойства не повторялись и не переопределяли друг друга в контексте одного css правила. Используйте минимальное количество кода для достижения цели.

Инструменты разработчика

Научитесь использовать плагины, позволяющее быстрее находить ошибки кода для каждого из браузеров.

  • Chrome - http://getfirebug.com/releases/lite/chrome/
  • Opera - http://www.opera.com/dragonfly/
  • Mozila - https://addons.mozilla.org/ru/firefox/addon/firebug/
  • Edge - https://docs.microsoft.com/en-us/microsoft-edge/f12-devtools-guide

Вендорные префиксы

Вендорные префиксы это приставки, используемые производителями (вендорами) браузеров для экспериментальных, еще не принятых в стандарт, CSS-свойств. Используйте инструменты для их автоматического добавления в код.

Автопрефиксер - http://autoprefixer.github.io/ru/

  • -webkit- (Chrome, Safari, newer versions of Opera.)
  • -moz- (Firefox)
  • -o- (Old versions of Opera)
  • -ms- (Internet Explorer)
  • -webkit-transition: all 4s ease;
  • -moz-transition: all 4s ease;
  • -o-transition: all 4s ease;
  • -ms-transition: all 4s ease;
  • transition: all 4s ease;

Выберите единицы в которых вы будете задавать размеры шрифтов на странице. Чем пользоваться это личный выбор каждого.

Рекомендованная статья для ознакомления с относительными единицами https://habrahabr.ru/post/280125/

Заключение

Понимание причин некорректного отображения сайта, а так же навык кроссбраузерной верстки приходит со временем и в процессе долгой практики. Только методом проб и ошибок человек может разобраться какие браузеры, какие заморочки имеют. С опытом, верстальщик старается составить код, так, чтобы он был всячески адаптирован под каждый из браузеров, а в процессе верстки оставалось только подправить небольшие косяки.
